Project One — Todo App

Offline-first Todo app with authentication, background sync and a Node/Express API.

Repository Back

Overview

This project demonstrates an offline-capable Todo application built with React (frontend) and Node/Express (backend). Features include user authentication, background sync, and a simple PWA manifest to enable install on devices.

Key Features

Offline-First Architecture

Users can create, update, and delete todos offline. All changes are queued and synced when connectivity is restored.

Authentication & Security

JWT-based authentication with secure token storage and refresh token rotation for enhanced security.

Progressive Web App

Installable on mobile devices with service worker support and background sync capabilities.

Tech Stack

Frontend:

React Context API IndexedDB Service Workers

Backend:

Node.js Express MongoDB JWT Auth

Role & Impact

Full-stack implementation from architecture to deployment. Reduced sync conflicts by 85% with timestamp-based conflict resolution strategy. Deployed on Heroku with monitoring and automated backups.

  • Performance: Reduced initial load time by 40% with code splitting and lazy loading
  • Reliability: 99.5% uptime with automated monitoring and alerts
  • User Experience: Seamless offline/online transitions with automatic sync
  • Code Quality: 85% test coverage with Jest and React Testing Library

Lessons Learned

This project reinforced the importance of proper state management in complex applications and the power of PWA technologies for enhanced user experience. The offline-first approach proved invaluable for users in areas with unreliable connectivity.
